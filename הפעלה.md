# מדריך הפעלת KRSTUDIO AI VISION

## דרישות מוקדמות

- Node.js (גרסה 16 ומעלה)
- npm או yarn
- Python 3.8+ (רק אם רוצים להפעיל את שרת ה-GPU)

## שלב 1: הגדרת משתני סביבה

1. עבור לתיקיית השרת:
```bash
cd server
```

2. צור קובץ `.env` מהדוגמה:
```bash
copy .env.example .env
```

3. ערוך את קובץ `.env` והוסף את ה-token שלך מ-Hugging Face:
```
HF_TOKEN=your_actual_token_here
```

## שלב 2: התקנת תלויות

### התקנת תלויות השרת:
```bash
cd server
npm install
```

### התקנת תלויות הקליינט:
```bash
cd client
npm install
```

## שלב 3: הפעלת המערכת

### אופציה א': הפעלה ידנית (מומלץ לפיתוח)

**טרמינל 1 - הפעלת השרת:**
```bash
cd server
npm run dev
```
השרת ירוץ על: `http://localhost:4000`

**טרמינל 2 - הפעלת הקליינט:**
```bash
cd client
npm run dev
```
הקליינט ירוץ על: `http://localhost:5174`

### אופציה ב': הפעלה בייצור

**השרת:**
```bash
cd server
npm start
```

**הקליינט (לאחר build):**
```bash
cd client
npm run build
npm run preview
```

## שלב 4: שימוש במערכת

1. פתח דפדפן וגש ל: `http://localhost:5174`
2. בחר בין יצירת תמונה או וידאו
3. כתוב פרומפט בעברית או באנגלית
4. לחץ על "צור" והמתן לתוצאה

## הערות חשובות

- **HF_TOKEN**: חובה להגדיר token מ-Hugging Face. ניתן לקבל [כאן](https://huggingface.co/settings/tokens)
- **פורטים**: 
  - שרת: 4000 (ברירת מחדל)
  - קליינט: 5174 (ברירת מחדל)
- **תיקיות**: המערכת יוצרת אוטומטית את התיקיות `outputs/` ו-`tmp/` אם הן לא קיימות

## פתרון בעיות

### שגיאת "Missing HF_TOKEN":
- ודא שקובץ `.env` קיים בתיקיית `server/`
- ודא שה-token מוגדר נכון בקובץ

### שגיאת CORS:
- ודא ש-`CLIENT_ORIGIN` ב-`.env` תואם לכתובת של הקליינט

### השרת לא מתחבר:
- ודא שהפורט 4000 פנוי
- בדוק את הלוגים בטרמינל
